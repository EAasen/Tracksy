## Summary
Generate a type-safe TypeScript/JavaScript API client from the OpenAPI specification to provide autocomplete, validation, and better developer experience.

## Current Status
- API client code written manually
- No type safety between client and server
- API changes require manual client updates
- No IDE autocomplete for API methods

## Acceptance Criteria

- `npm run generate:client` creates up-to-date client
- Generated client has full TypeScript types
- IDE autocomplete works for all API methods

- Frontend code uses generated client

- Documentation shows client usage examples

## Implementation Notes

### Recommended Tool: openapi-typescript-codegen
```bash
npm install --save-dev openapi-typescript-codegen
```

### Package.json Script
```json
{
  "scripts": {
    "generate:client": "openapi-typescript-codegen --input backend/openapi.yaml --output frontend/src/api/generated --client fetch"
  }
}
```

### Usage Example
```typescript
import { AuthService } from '@/api/generated';

// Type-safe API call with autocomplete
const response = await AuthService.login({
  email: 'user@example.com',
  password: 'password123'
});
// response is typed as LoginResponse
```

### Alternative Tools
- **openapi-typescript**: Types only, minimal runtime
- **@hey-api/openapi-ts**: Modern, lightweight
- **openapi-generator**: Java-based, feature-rich but heavy
