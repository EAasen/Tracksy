## Summary
Split the monolithic OpenAPI specification into modular files for better maintainability and team collaboration using external references.

## Current Status
- Single file: `backend/openapi.yaml` (~500-1000 lines)
- All schemas, paths, and components in one file
- Difficult to navigate and maintain
- Merge conflicts likely as spec grows

## Tasks
- [ ] Create modular directory structure
- [ ] Split schemas into individual files
- [ ] Split paths by resource (auth, activities, health-metrics)
- [ ] Use $ref for external references
- [ ] Create build script to bundle for serving
- [ ] Update package.json with build commands
- [ ] Update Docker setup to serve bundled spec
- [ ] Test Swagger UI with bundled spec
- [ ] Update documentation

## Proposed Structure
```
backend/
  openapi/
    openapi.yaml           # Main entry point
    components/
      schemas/
        Error.yaml
        User.yaml
        Activity.yaml
        HealthMetric.yaml
        Goal.yaml
      responses/
        Error400.yaml
        Error401.yaml
        Error404.yaml
        Error500.yaml
      parameters/
        UserId.yaml
        ActivityId.yaml
    paths/
      auth.yaml
      activities.yaml
      health-metrics.yaml
      health.yaml
```

## Acceptance Criteria
- OpenAPI spec split into logical modules
- Main openapi.yaml uses $ref for all external files
- Build script bundles into single spec for serving
- `npm run build:openapi` creates bundled spec
- Swagger UI renders correctly from bundled spec
- All existing endpoints still documented
- Developer docs explain modular structure

## Implementation Notes

### Tools
- @apidevtools/swagger-cli for bundling
- Install: `npm install --save-dev @apidevtools/swagger-cli`
- Bundle: `swagger-cli bundle openapi/openapi.yaml -o openapi.yaml -t yaml`

### Example $ref Usage
```yaml
# openapi/openapi.yaml
paths:
  /api/v1/auth:
    $ref: './paths/auth.yaml'
  /api/v1/activities:
    $ref: './paths/activities.yaml'
components:
  schemas:
    Error:
      $ref: './components/schemas/Error.yaml'
```
