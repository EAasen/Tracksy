## Summary
Add custom `x-changelog` OpenAPI extensions to track breaking changes, deprecations, and version history directly in the API specification.

## Current Status
- No machine-readable change tracking in OpenAPI spec
- CHANGELOG.md exists but not linked to spec
- No deprecation warnings in spec
- Difficult to generate version-specific documentation

## Rationale
x-changelog extensions provide:
- Machine-readable version history
- Automatic deprecation notices in docs
- Breaking change tracking per endpoint
- API evolution visibility
- Generated migration guides

## Tasks
- [ ] Design x-changelog extension schema
- [ ] Add x-changelog to top-level OpenAPI spec
- [ ] Add x-deprecated extension to deprecated endpoints
- [ ] Add x-version-added to new endpoints
- [ ] Document extension format in CONTRIBUTING.md
- [ ] Create script to validate changelog consistency
- [ ] Update Swagger UI to display changelog
- [ ] Generate CHANGELOG.md from OpenAPI spec
- [ ] Add examples to all documented endpoints

## Acceptance Criteria
- x-changelog defined with clear schema
- All breaking changes documented in spec
- Deprecated endpoints marked with x-deprecated
- Swagger UI shows deprecation warnings
- Script validates changelog entries
- CHANGELOG.md auto-generated from spec
- Documentation explains extension usage

## Implementation Notes

### x-changelog Extension Schema
```yaml
info:
  x-changelog:
    - version: 1.0.0
      date: 2024-01-15
      changes:
        - type: added
          description: Initial API v1 release
    - version: 1.1.0
      date: 2024-02-01
      changes:
        - type: changed
          description: Standardized error responses
          breaking: false
        - type: deprecated
          description: /api/activitylog endpoint (use /api/v1/activities)
          breaking: false

paths:
  /api/v1/auth/login:
    x-version-added: "1.0.0"
    post:
      summary: User login
      
  /api/activitylog:
    x-deprecated:
      since: "1.1.0"
      removeIn: "2.0.0"
      useInstead: /api/v1/activities
      description: Legacy endpoint, migrate to v1 API
    get:
      deprecated: true
      summary: Get activities (deprecated)
```
